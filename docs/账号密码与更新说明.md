## 4. 主要更新内容

### 4.1 Docker 部署修复

- 修复前端 Docker 构建镜像版本，改用 Node 22 Alpine，避免新版 Vite 在旧 Node 版本下构建失败。
- 新增 `frontend/.dockerignore`，避免 Docker 构建上下文带入 `node_modules`、`dist` 等本地文件。
- 调整 `docker-compose.yml`：
  - Nacos 等待 MySQL 健康后再启动。
  - Nacos healthcheck 改为 `/nacos/actuator/health`。
  - Seata 镜像升级为 `seataio/seata-server:2.0.0` 并增加健康检查。
  - 网关增加失败重启策略。
  - 订单、库存、基础服务增加 Seata 相关环境变量。
  - 关闭业务服务对 Nacos 配置中心的强依赖，避免配置为空时启动失败。

### 4.2 数据库初始化修复

- SQL 初始化脚本补充 `CREATE DATABASE` 和 `USE`，让 MySQL 容器首次启动时能自动建库。
- 为 `base_service_db`、`inventory_service_db`、`order_service_db` 增加 Seata `undo_log` 表。
- 修复多库初始化顺序和 Docker Compose 初始化兼容性。

### 4.3 菜单和权限修复

- 新增统一前端角色判断工具 `frontend/src/utils/access.ts`。
- 侧边栏菜单按角色动态显示：
  - 管理员显示全部菜单。
  - 普通用户只显示首页。
  - 销售、采购、仓库、财务分别只显示对应业务菜单。
- 路由层增加角色 guard，用户即使手动输入 URL，也不能打开无权限页面，会被重定向回首页。
- 修复普通用户误看到订单管理、基础信息管理后点击出现 403/Access Denied 的问题。
- 库存服务的分类和仓库接口增加后端权限控制，避免直接请求接口绕过前端限制。

### 4.4 前端缓存修复

- 调整 `frontend/nginx.conf`：
  - `/index.html` 和前端路由入口禁用缓存。
  - `/assets/**` 使用长期缓存。
- 解决前端重新构建后，浏览器仍加载旧菜单代码的问题。

### 4.5 对外端口收敛

- `docker-compose.yml` 默认只对宿主机开放：
  - `127.0.0.1:18080 -> frontend:80`
  - `127.0.0.1:18848 -> nacos:8848`
- 以下服务不再映射到宿主机端口，仅在 Docker 内网通信：
  - MySQL `3306`
  - Seata `8091`
  - Gateway `9090`
  - user-service `8081`
  - role-service `8082`
  - order-service `8083`
  - inventory-service `8084`
  - base-service `8085`
- 如果本机 `18080` 或 `18848` 也被占用，可以启动前临时指定端口：

```bash
FRONTEND_PORT=28080 NACOS_PORT=28848 docker compose up -d
```

### 4.6 Docker 一键构建

- 后端服务改为统一的 `backend/Dockerfile` 多阶段构建。
- `docker compose up -d --build` 会在 Docker 构建阶段自动执行 Maven 打包，不需要提前把 `target/*.jar` 放进源码包。
- Docker 构建阶段使用 `backend/maven-settings.xml` 配置 Maven 镜像源，降低国内网络下拉依赖失败的概率。
- 本地仍可执行 `mvn clean package -DskipTests` 做开发调试，但 Docker 部署不强依赖这一步。
